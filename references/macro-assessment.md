# 宏观评估体系（4类）

## #1 四维度论文评估

### 评估框架

评估学术论文整体质量的四个核心维度：

| 维度 | 权重 | 核心判据 | 满分 |
|------|:----:|---------|:----:|
| 内容完整性 | 0.30 | 模块覆盖度、章节完备性、关键要素无遗漏 | 100 |
| 逻辑完整性 | 0.25 | 章节间逻辑链连贯、前后一致、无断层 | 100 |
| 逻辑闭环 | 0.25 | 摘要承诺→正文展开→结论回收的完整闭环 | 100 |
| AI检测评估 | 0.20 | AI生成痕迹占比、语言净化程度 | 100 |

### 加权评分公式

总分 = 内容完整性×0.3 + 逻辑完整性×0.25 + 逻辑闭环×0.25 + AI检测评估×0.2

### 优质线

85分以上具备送审条件。

### 操作流程

1. 通读全文，建立内容地图（章节→小节→关键要素）
2. 逐维度打分，标注扣分原因
3. 交叉验证：逻辑链断裂点是否已被内容覆盖
4. 输出加权总分 + 四维度分项分数
5. 修正预估分

## #2 评分演进追踪

### 目的

跨版本追踪论文质量变化趋势，识别改进方向和退化风险。

### 追踪维度

- 每次审查的版本号、评分、关键发现
- P0/P1/P2/P3各等级问题数量变化
- 新增问题 vs 已修复问题
- 退化项（修复引入的新问题）

### 输出格式

```
v11(71分) → v19(78-82分) → v20.16(82分) → V3.0.2(86.3分) → V3.0.7(89.1分) → 目标85+
```

### 约束

- 分数必须单值，禁止模糊区间（如"75~80分"）
- 每次评分须标注评分依据（基于哪些维度的什么发现）

## #3 19项审视法

### 方法结构

对论文进行结构化逐一审视，每一项给予：

1. 编号（1-19）
2. 严重性判定（P0/P1/P2/P3）
3. 修改建议
4. 可撤销/可降级标记

### 自纠机制

- 允许**撤销**错误判定（如AI痕迹过度敏感可撤销）
- 允许**降级**过度反应（如长句从P2降为P3）
- 输出格式：「19项问题中：撤销1项/降级1项/保留17项（P0级4项/P1级6项/P2级3项/P3级3项）」

### 适用范围

- 论文初稿首次全面审视
- 重大版本重构后复查

## #4 国家级专家三步审查法

### 适用场景

多版本选型决策（如V3.0.0 vs V3.0.1 合并/选型）。

### 三步流程

**第一步：详细差异列表并评判优劣**
- 逐项列出两版本所有差异
- 每项归属优劣方
- 标注差异性质（内容新增/删除/改写/架构调整）

**第二步：分别审计各版本对原始需求的满足度**
- 各自是否满足7项核心要求
- 各自是否落实14项已确认决策
- 数据诚信度对比

**第三步：结构化Markdown综合输出**
- 合并方案推荐
- 最优底本选择 + 迁移项清单
- 拒绝回退项清单

### 输出格式

| 对应章节 | V3.0.0 | V3.0.1 | 差异判定 | 优劣归属 |
|---------|--------|--------|---------|---------|
| Ch1 | xxx | xxx | 内容一致 | — |
| Ch2 | xxx | xxx | 重写 | V3.0.1优 |

### 典型结论范式

「两个版本均无法独立作为终稿。V3.0.0数据诚信度最高但参考文献有系统性错误；V3.0.1写作质量与模块框架最优但Ch2存在编造数据硬伤。」
