# 专项审查框架（8类）

## #8 插图引用审查

### 四维检查

1. **引用存在性**：每张图是否有至少1次正文引用
2. **引用句式**：是否存在"如图X所示""见图X"等显式引导语
3. **引用位置**：引用是否在图首次出现前/附近
4. **编号连续性**：图编号是否按正文出现顺序连续（1→2→3→4→...）

### P0级判定条件

- 某图全文零次引用 → **P0硬伤，必删**
- 图编号出现跳跃/不连续 → **P0硬伤，重排**
- 引用指向错误（"如图3"实际描述图5内容）→ **P0硬伤**

### 输出格式

```
图1 ✅ 引用2次（P027、P144）编号位置正确
图2 ✅ 引用1次（P087）编号正确
图3 ✅ 引用1次（P095）编号正确
图4 ✅ 引用1次（P103）编号正确
图5 ❌ 全文零引用 → P0硬伤，建议删除
```

## #9 图文割裂分析（三维度判据）

### 三维度

- **维度一：引图对象错配** — "如图X所示"所描述的内容是否与图X主题一致
- **维度二：图文断裂** — 图片是否独立成段打断正文流；后续小节是否回引
- **维度三：缺乏过渡句** — 图片前后是否有文字衔接和解释

### 判定标准

- 三个维度**均不合格** → 建议删除该图
- 两个维度不合格 → P1级，需加强衔接或考虑删除
- 一个维度不合格 → P2级，补充过渡句即可

## #10 插图删减可行性分析

### 核心判据

**该图是否提供正文无法替代的信息增量？**

- 是 → 保留（如架构图、数据流程图）
- 否 → 可删（如与正文高度重叠的插图）

### 三档方案

| 方案 | 保留数量 | 筛选标准 |
|------|:------:|---------|
| 保守 | 7-8张 | 凡是有效图片均保留 |
| 推荐 | 6张 | 每图提供不可替代的信息增量 |
| 激进 | 5张 | 仅保留核心架构和数据图 |

### 输出范例

「推荐保留6张：图1架构/图2四码联动/图3工序追溯/图6成效/图7AI/图8排产」
「图5必删理由：无正文引用硬伤，与图3功能重叠」

## #11 图题规范检查

### 检查项

1. 图题是否准确反映图内容（不溢出、不遗漏）
2. 图题是否过细（不应包含模块枚举、技术参数）
3. 图题格式是否规范（居中、字号、加粗）
4. 图题是否有句号（学术规范：图题不加标点）

### 典型修正

- 「图1 溯源系统四层闭环架构及十大核心模块」→「图1 溯源系统四层闭环架构」
- 「图2 系统从销售报价到成品发货的全流程业务闭环图」→「图2 业务全流程审批与数据流转」

## #12 段落级归属修正检查

### 目的

核对每个模块/功能的描述段落实际所在章节，与其逻辑归属是否一致。

### 检查方法

1. 提取所有二级/三级标题
2. 逐模块标注其"宣称归属"（摘要/前言所述）
3. 逐一核对其"实际位置"（正文所在章节）
4. 标记归属不一致的段落

### 典型案例

- 设备管理（实际Ch4）和报价主数据（实际Ch5）被误归入基础支撑层（Ch3）
- 修复：移动段落或修正摘要中的归属描述

## #13 AI语言审查框架（A/B/C三级）

### 三级分类

| 级别 | 特征 | 典型词汇/句式 | 处理方式 |
|------|------|-------------|---------|
| **A级** | AI痕迹明显 | 四"自"（自感知/自学习/自优化/自适应）、绝对化断言（"彻底消除""根除"）、拟人隐喻（"永不懈怠""永不犯错"）、营销措辞（"核心竞争壁垒""根本性颠覆"） | **必须改写** |
| **B级** | 术语偏营销但有数据支撑 | "秒级追溯""7×24小时""全量覆盖" | 可保留但需数据锚定 |
| **C级** | 技术描述具体 | 工程术语、量化指标、操作流程 | 直接保留 |

### 核心转换范式

| AI痕迹表达 | 学术改写 |
|-----------|---------|
| 自感知 | 动态感知 |
| 自学习 | 持续优化 |
| 自优化 | 自动调参 |
| 自适应 | 动态匹配 |
| 永不懈怠 | 持续运行 |
| 协同进化 | 联合优化 |
| 彻底消除 | 有效规避 |
| 根本性颠覆 | 实质性改进 |
| 从0到1的关键突破 | 完成了可行性验证 |
| 第四次工业革命 | 新一代信息技术变革 |
| 毫秒级响应 | 实时响应（无测量依据时不写量级） |

### 操作流程

1. 全文逐段标记A/B/C三级
2. 统计各级占比
3. 对A级逐条改写
4. 输出净化结果报告

## #14 模块数量一致性检查

### 方法

1. 统计全文所有顶层模块的实际数量（Heading 2级别）
2. 搜索论文中所有宣称模块数量的位置（如"十大核心模块"）
3. 发现不一致 → 选择：调整结构使其一致 / 修正宣称数

### 话术模板

「论文四处宣称"十大核心模块"，但实际统计为11处：模块A、B、C... → 需合并或修正宣称」

## #15 全文涟漪扫描

### 目的

在每次修改后，以修改点为圆心向外扫描，确认修改是否产生涟漪效应（影响其他段落）。

### 扫描范围

全文所有段落 + 所有表格 + 所有图片。

### 任务

1. 确认每条修改是否产生涟漪效应
2. 明确标记"无需修改"的段落
3. 输出精确的修改段落编号列表

### 输出格式

```
P014引言 ✓无影响
P026节标题 ✓无影响
P129数据源 ✗需修改
P217结论 ✓无影响
```
