# 数据与合规验证（7类）

## #19 数据真实性验证

### 核心原则

**严禁编造数据。所有引用必须可溯源确认。**
所有数据引用须满足至少一项：系统实际运营数据 / 可查证行业统计数据 / 标注来源的文献引用。

### 验证手段

| 数据类别 | 验证方法 | 可信度 |
|---------|---------|:----:|
| 系统内部数据 | 交叉核对系统截图、多个版本一致性 | 高 |
| 行业统计数据 | 追溯来源文献、核实年份 | 中 |
| 声称数据 | "超过200家"等模糊断言须有可靠来源 | 低 |
| 成效数据 | 核对原文多处一致性、趋势合理性 | 中高 |

### 不可确认数据的处理

- 使用"[需补充]"占位符
- 行业现状章节等可交由外部专家补充
- 绝对不编造任何数据

### 判定话术

- 「该数据来源不可信，建议替换为[需补充]占位符」
- 「系统实际运营数据，来源可确认」— 可信数据标注

### 行业统计数据时效性检查

当论文引用行业宏观数据（如"持证企业逾1.6万家"）时，须同时验证：
1. **年份是否最新**：是否有更新的官方通报可用
2. **数据是否可精确验证**：模糊数字（"逾1.6万"）能否替换为官方精确数字（"16977张"）
3. **分解数据是否可证实**：如"压力容器制造企业超5000家"——市监总局通报从未按设备类型分解制造许可数量，此类分解数字无法验证时应改为定性表述

### 典型案例

```
原文: "截至2022年底，全国持有特种设备制造许可证的企业逾1.6万家，
       其中压力容器制造企业超过5000家"

验证:
  - "2022年底" → 2024年度通报已发布（2025-05-07），应更新年份
  - "逾1.6万家" → 官方精确数字为"制造许可证16977张"（2024年）
  - "压力容器制造企业超过5000家" → 官方从未发布此分解数据，无法验证

修正:
  "截至2024年底，全国共颁发特种设备制造许可证16977张，
   压力容器为其中占比最大的品类之一"
```

## #20 数据双抱归因分析

### 核心概念

**双抱归因**：同一对数字/同一效果被归因于两个不同的阶段、系统或原因。

### 检测方法

1. 全文搜索所有量化数据点
2. 为每个数据点建立全文出现位置索引
3. 检查同一数据在不同章节的归因对象
4. 发现不一致 → 输出分析报告

### 典型案例

报价时效"2h→10min"同时出现在：
- P129：归因于"数字化系统已实现"
- P193：归因于"AI智能体有望实现"

→ 同一对数字归因于两个不同阶段，逻辑矛盾。

### 修正方案

- **推荐**：改为增量表述（10min→2min），避免双抱归因
- **备选**：仅保留一处（归因最准确的位置），删除另一处

## #21 中英文摘要对称性检查

### 检查矩阵

| 检查项 | 方法 | P级 |
|-------|------|:---:|
| 模块数量对称 | 中文"十大" vs 英文数量 | P1 |
| 关键词一一对应 | 逐词比对 | P1 |
| 核心数据对称 | 成效数据是否中英文一致 | P0 |
| AI/技术方向术语 | 对称性和翻译准确性 | P2 |
| 字数/篇幅对称 | 中英文长度比例合理 | P3 |

### 典型发现

「中文摘要缺失4项数据（设备报修/沟通成本/复购率/合规），英文摘要已包含 → 不对称 → P1」

## #22 参考文献逐条比对

### 五项检查

1. **正文引用标注**：每条参考文献是否在正文中至少出现1次引用标注
2. **格式统一性**：是否符合GB/T 7714 / APA等指定格式
3. **编号连续性**：[1][2]...[N]是否连续无跳跃
4. **占位符检查**：是否存在"张三、李四""待补充"等占位符
5. **标准文献版本**：引用标准的年份版本是否正确（如GB/T 47120-2026）

### P0判定条件

- 某条参考文献全文零次引用 → **P0硬伤**
- 参考文献含占位符姓名 → **P0硬伤**
- 编号不连续 → **P1**

### 输出格式

```
[1] ✅ 引用1次 ✓格式正确
[2] ✅ 引用3次 ✓格式正确
...
[10] ❌ 零次引用 → P0硬伤
[14] ❌ 零次引用 → P0硬伤
[17] ❌ 零次引用 → P0硬伤
```

## #23 脱敏筛查

### 核心原则

**技能/论文/报告公开发布前，必须进行逐项脱敏审查。** 确保不泄露：人员身份信息、组织内部数据、未公开论文内容、API凭证等敏感信息。

### 五级脱敏检查清单

| 级别 | 检查项 | 判定标准 | P级 |
|:----:|--------|---------|:---:|
| 🔴 L1 | **人员姓名** | 作者、导师、同事、用户真实姓名 | P0 |
| 🔴 L1 | **组织名称** | 公司全称、学校名称、部门名称 | P0 |
| 🔴 L1 | **凭证/Token** | API Key、Token、密码、密钥 | P0 |
| 🟠 L2 | **论文完整标题** | 含指向性关键词的论文全称 | P1 |
| 🟠 L2 | **具体数据数字** | 订单数、产品数、扫码次数等精确运营数据 | P1 |
| 🟡 L3 | **产品/项目代号** | 内部系统名称、模块代号、版本命名 | P2 |
| 🟡 L3 | **引用标准编号** | 公开发布的国家/行业标准（GB/T、NB/T等） | 🟢 安全 |
| 🟢 L4 | **通用技术术语** | 如"四码联动""微服务架构""RBAC" | 🟢 安全 |
| 🟢 L4 | **方法论/框架名称** | 通用架构模式、设计模式 | 🟢 安全 |
| ⚪ L5 | **评分/成绩数字** | 抽象评分（如"71→91.5分"）无指向具体对象 | 🟢 安全 |

### 筛查流程

1. **全文扫描** → 搜索模式匹配：姓名、公司名、数字、特殊编码格式
2. **上下文判定** → 区分"通用概念"与"指向性信息"（同一术语在不同上下文中风险不同）
3. **脱敏处理** → 删除 → 通用化改写 → 模糊化（保留量级、移除精确值）
4. **二次复检** → 独立审查脱敏后版本，确认无遗漏

### 常见高风险模式

| 模式 | 正则参考 | 风险 |
|------|---------|:---:|
| 中文姓名 | `[\u4e00-\u9fa5]{2,4}(?=先生|女士|老师|教授)` | 🔴 |
| 论文标题 | `《.*》` 或引号内长句含领域关键词 | 🟠 |
| Token格式 | `skh_[a-f0-9]{64}` / `sk-...` | 🔴 |
| 内部编号 | 纯数字序列（4-6位）配合内部业务术语 | 🟡 |
| 标准编号 | `GB/T \d+` `NB/T \d+` `ISO \d+` | 🟢 |

### 判定话术

- 「该处含作者真实姓名 → L1/P0 → 删除」
- 「论文完整标题含行业+公司指向 → L2/P1 → 通用化改写」
- 「技术术语无指向性 → L4/安全 → 保留」
- 「脱敏审查完成：L1×0 / L2×1（已改写）/ L3×2（安全）/ L4×5（安全）」

### 与技能发布的关系

技能（Skill）公开发布前**必须**完成脱敏筛查。本方法论自身即典型案例：
v1.0.0 发布时 README 含论文完整标题（L2级），v1.0.1 改为"an engineering master's thesis on industrial digitalization"后脱敏通过。

### 自纠示例

```
审查发现：README.md 中含论文完整标题 "Design and Application of..."
↓ 上下文判定：L2级（含行业+应用领域指向）
↓ 处理：通用化改写，删除论文标题，改为"an engineering master's thesis on industrial digitalization"
↓ 复检：✅ 无人员/组织/数据泄露风险
```

## #32 参考文献真实性交叉验证

### 背景

多轮修改中新增或替换的参考文献，可能存在**作者名/期刊名/年份/卷期号与数据库记录不一致**的系统性错误，甚至**整条文献完全编造**（常见于AI辅助写作场景）。此类问题对于论文投稿是致命性的——审稿人只需查一条不存在文献即可否定整篇论文的学术诚信。

### 检测方法

1. **逐条搜索**：对每条参考文献在知网/万方/百度学术中搜索（优先使用DOI直查）
2. **核心字段核对**：逐一比对以下字段与数据库记录是否一致

| 核对字段 | 常见错误模式 | P级 |
|---------|-------------|:---:|
| **作者** | 改名（Li X→Bouhala L）、缺人（等→补全3人）、假名 | 🔴 P0 |
| **期刊** | 降级/升级/全错（"计算机工程与设计"→"江苏工程职业技术学院学报"） | 🔴 P0 |
| **年份** | 年份错（2019→2024） | 🔴 P0 |
| **卷期** | 卷/期号不匹配 | 🟡 P1 |
| **DOI** | 存在但不可解析 | 🟠 P0 |
| **标题** | 微调措辞 | 🟡 P1 |

3. **分类判定**：
   - **期刊错/年份错 → 修正**（文献存在但元数据错误）
   - **完全搜不到 → 编造文献 → 替换**（用同主题真实文献替换）

### 编造文献的典型特征

- 作者组合不可查（如"李贵+丁子舾+彭钦銮"在知网/百度学术中完全不存在）
- 英文文献作者名为中文名拼音（Li X, Zhang Y）而非真实作者名（Bouhala L, Perbal S）
- 期刊卷期与发表年份不匹配
- 标题在对应期刊的对应年份卷期中完全搜不到

### 操作流程

```
1. 导出所有参考文献条目 → 提取：作者、标题、期刊、年份、卷期、DOI
2. 逐条在数据库中搜索（优先级：DOI > 标题+作者 > 作者+期刊+年份）
3. 核对6项核心字段
4. 输出分类报告：
   - ✅ 完全匹配
   - ✏️ 需修正（期刊/年份/作者等有误）
   - 🔃 需替换（编造文献，提供真实替代方案）
5. 执行修正/替换 → 重新编号（触发#31引用重编号）
```

### 输出格式

```
参考文献真实性审查报告
========================
[1] 陈学东等. 机械工程学报, 2023 → ✅ 完全匹配
[7] 王勇等. 计算机工程与设计, 2019 → ✏️ 需修正
    实际: 江苏工程职业技术学院学报, 2024, DOI: 10.19315/j.issn.2096-0425.2024.03.004
[13] 李贵等. 制造业自动化, 2025 → 🔃 编造文献，需替换
    建议: 阮斯洁等. 计算机集成制造系统, 2023 (EI核心，被引34次)
[17] Li X, Zhang Y. Machines, 2026 → 🔃 编造作者名
    实际: Bouhala L, Perbal S. J. Compos. Sci., 2026, DOI: 10.3390/jcs10030171

汇总: ✅4条 / ✏️2条 / 🔃2条
```

### P0级判定条件

- 某条文献在数据库中完全搜不到 → **P0致命**（编造文献，必须替换）
- 英文文献作者名为中文名拼音（Li X, Zhang Y）→ **P0致命**（AI杜撰假作者）
- 期刊名完全错误 → **P0**（需修正）
- 作者缺人但核心作者在 → **P2**（需补全但不致命）

### 经验笔记

- V3.3.9→V3.3.13实战：18条参考文献中发现2条编造（[13]李贵等/[17]Li X et al.）+ 3条期刊/年份错误（[7][11][15]）
- 编造文献在AI辅助写作中极其常见——AI倾向于"补全"参考文献部分而非留空
- 唯一英文文献[17]必须重点审查：审稿人首先验证的就是外文文献
- 文献修正后必然触发引用重编号——#32与#31紧密关联

## #33 规范性文件引用缺失检测

### 背景

论文正文中提及的国家标准（GB、GB/T）、行业标准（NB/T、JB/T）、安全技术规范（TSG）、行政法规（总局令）等规范性文件，应使其在参考文献列表中拥有对应的引用角标。但实践中经常出现：正文大段论述基于某项标准展开，该标准却**未挂载任何引用角标**，读者无法追溯合规依据。此类问题在自动化审查中极易漏检，因为审查工具看到的是"文本提及了标准编号"而非"该标准缺少引用角标"。

### 检测方法

1. **全文扫描规范性文件文本提及**：搜索正则模式 `GB[/T]? \d+` `NB/T \d+` `TSG [A-Z]\d+` `总局令第\d+号` `第\d+号令` `ISO \d+` 等
2. **提取提及位置**：记录每个提及所在的段落及上下文
3. **审计角标挂载状态**：逐条检查该标准/法规提及处是否已挂载引用角标
4. **判定缺失**：
   - 若该标准在正文中被实质性讨论（非仅名称罗列）但无引用角标 → **P0**
   - 若该标准仅提及但未展开讨论 → **P1**（建议补充）

### 常见缺失模式

| 正文提及 | 引用状态 | 问题 |
|---------|:---:|------|
| "系统基于GB/T 150.1~150.4" | ❌ 无角标 | 全文依赖此标准但无引用 |
| "NB/T 47014《承压设备焊接工艺评定》" | ❌ 无角标 | 焊接Q&A核心标准缺失引用 |
| "NB/T 47015《压力容器焊接规程》" | ❌ 无角标 | 同上 |
| "73号令明确要求……" | ✅ 有[10] | 引用正确 |

### 操作流程

```
1. 全文正则扫描 → 提取所有规范性文件提及（标准编号+法规编号）
2. 去重后逐条检查正文提及处是否已挂载引用角标
3. 分类输出：
   - ✅ 已引用（标注引用编号）
   - ❌ 未引用但正文实质性讨论（P0，需补充）
   - ⚠️ 仅名称提及（P1，建议考虑是否补充）
4. 对P0项：判断是否需要新增参考文献条目，或合并到已有文献引用中
```

### 输出格式

```
规范性文件引用审计报告
========================
GB/T 150.1~150.4-2024 → [15] ✅ 已引用
GB/T 25198-2023 → [16] ✅ 已引用
NB/T 47014 焊接工艺评定 → ❌ 正文大段讨论但无引用角标 (P0)
NB/T 47015 压力容器焊接规程 → ❌ 同上 (P0)
73号令 → [10] ✅ 已引用，3次
GB/T 47120-2026 → [11] ✅ 已引用

汇总: ✅5项 / ❌2项 (P0)
```

### P0级判定条件

- 正文依赖某项标准但该标准在参考文献中无条目且无引用角标 → **P0**
- 正文引用标准编号但版本标注错误（如GB/T 150-2011而最新版为2024）→ **P0**

### 经验笔记

- NB/T 47014 和 NB/T 47015 在论文V3.0.5前长期缺失引用，是焊接质量归因修正（参数迭代→WPS合规执行）后的衍生检查项
- 规范性文件的标准编号写法须精确："GB/T 150.1~150.4"而非"GB 150"、"TSG Z6002-2010"而非"TSG Z6002"
- 部分标准（如73号令）可能同时实质性讨论+仅名称提及，提取时合并同一标准的多次出现
- #33 通常在 #32（文献真实性验证）之后执行，因为文献修正可能影响引用编号
