# 需求满足度审计（3类）

## #24 14项决策审计

### 审计方法

以论文写作过程中已确认的14项决策为审计基准，逐一核查当前版本是否落实。

### 典型决策项示例

| # | 决策内容 | 核查方式 |
|---|---------|---------|
| 1 | AI融合功能拆分（已落地入Ch5/Ch6、规划入Ch7） | 全文检索AI相关段落归属 |
| 2 | 模块描述原则性模糊化 | 抽样检查模块段落粒度 |
| 3 | 删除工序级细节描述 | 全文搜索工序名 |
| 4 | AI内容独立章节 | 检查章节结构 |
| 5 | 问题链驱动叙事 | 引言→正文逻辑链检查 |
| 6 | 设计理念独立为独立章 | 章节结构核对 |
| 7 | 保留十大模块+四层闭环 | 全文架构术语统一 |
| 8 | 成效数据双轨制（已实现+规划） | 成效描述时态审查 |
| 9 | Ch4架构师视角、Ch5工程师视角 | 视角一致性检查 |
| ... | ... | ... |

### 输出格式

```
14项决策审计：
✅ 满足：10/14
⚠️ 部分满足：3/14
❌ 未满足：1/14
```

## #25 7条专家需求满足度审计

### 审计目标

以国家级论文专家审查后提出的7条核心改进要求为基准，逐条核查。

### 审计项（示例）

| # | 专家需求 | 核查标准 |
|---|---------|---------|
| 1 | 问题链驱动叙事结构 | 引言是否有明确的问题递进链 |
| 2 | 五大类别归并（非削减） | 模块归类是否完整覆盖 |
| 3 | 删除工序级描述，仅保留模块职责与边界 | 全文是否残留工序名/步骤码 |
| 4 | AI入第7章（应用成效与前景） | AI内容章节位置确认 |
| 5 | 设计理念独立为第3章 | 章节结构核对 |
| 6 | 模块改写结构："设计思想→实现机制→应用成效" | 模块段落的三段式结构 |
| 7 | 系统截图不放论文正文 | 全文图片来源检查 |

### 输出格式

「V3.0.1对7条要求的满足度：完全满足×/7 部分满足×/7 未满足×/7」

## #26 安全底本策略

### 策略逻辑

当存在多个版本各有优劣时，采用安全底本策略生成最佳合并版：

1. **选定安全底本**：以数据诚信度最高、无编造数据的版本为底本
2. **迁移改进项**：从其他版本迁移写作质量、架构组织等改进
3. **拒绝回退**：其他版本中的编造数据/占位符等不回退到底本
4. **生成合并版**：底本 + 迁移项 = 最优版本

### 优先级原则

- 数据诚信度 > 写作质量 > 架构美观度
- 可溯源数据 > 不可溯源数据
- 工程真实描述 > AI美化表述

### 典型话术

「安全底本策略：以数据诚信度为最高优先级，从最优版本中迁移写作质量改进，拒绝回退编造数据。」
