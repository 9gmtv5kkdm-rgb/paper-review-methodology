# 问题分级体系（3类）

## #5 P0/P1/P2/P3 四级严重性分类

### 分级标准

| 级别 | 标签 | 含义 | 典型示例 |
|------|------|------|---------|
| **P0** | 🔴 硬伤/阻断 | 写错的事实、逻辑矛盾、数据编造、必删必改 | Ch2编造"超过200家"、图5全文零引用 |
| **P1** | 🟡 高优 | 架构缺陷、模块遗漏、归属错误 | 十大模块遗漏物料仓储模块、三层架构命名混乱 |
| **P2** | 🟠 中等 | 完整性建议、数据补全、措辞优化 | 结论缺关键成效数据、某些模块描述未展开 |
| **P3** | 🟢 低优/可选 | 措辞统一、细微调整、个人偏好 | "质量管理驾驶舱"→"质量数据可视化看板" |

### 判定话术

- **P0判定**：「如果这条不改，论文送审会被直接指出问题」
- **P0特征**：未经任何讨论就必须修正
- **P3降级前提**：「仅在自然撰写时顺手优化，不作为独立修改项」

### 操作规范

- 任何P0判定必须给出明确理由（为何不修不行）
- 允许争议：若用户有不同判断，P0可降级
- 禁止模糊分级：不允许"P0/P1边缘"这类表述

## #6 硬伤/软伤分类法

### 硬伤类型清单

以下是论文写作中常见的硬伤类型，凡命中即为P0或P1级：

1. **交叉引用错误**：章节号/图号/表号引用指向错误目标
2. **中英文摘要数据不一致**：中文和英文摘要中的核心数据不对称
3. **时态与章节定位不符**：已有功能用将来时、规划功能用现在时
4. **关键词未同步**：中英文关键词不一一对应
5. **模块数量不一致**：全文多处宣称的模块数与实际统计不符
6. **图文引用缺失**：图片/表格全文零次正文引用
7. **数据双抱归因**：同一数据归因于两个不同阶段/原因
8. **参考文献零引用**：某条参考文献在正文中没有任何引用标注
9. **占位符残留**："张三、李四""[待补充]"等未清除
10. **编造数据**：无来源、不可溯源的数据点

### 软伤类型

- 措辞过度（"彻底消除""根本性颠覆"）
- 句式模板化（排比句堆砌）
- AI痕迹明显（四"自"句式等）
- 营销语言（"核心竞争壁垒"等投资人词汇）
- 缺少过渡句
- 段落过长无拆分

### 判定原则

- 硬伤→P0/P1，必须修复
- 软伤→P2/P3，视情况处理

## #7 数据涟漪效应分析

### 核心概念

论文中任何一个数据点的修改都可能引发跨章节的涟漪效应。分析任务是：当识别到某数据错误后，追踪该数据在全文所有出现位置，评估修改它的连锁影响。

### 操作流程

1. **全文结构化提取**：逐段编号，提取所有数据点
2. **数据点去重与索引**：同一数据在不同位置的出现建立索引
3. **涟漪路径分析**：识别修改目标数据后会影响的段落
4. **相关论述影响评估**：数据变化是否导致结论/归因/论述需要调整
5. **输出涟漪分析报告**：标记"需修改"和"不受影响"的段落

### 典型案例：双抱归因

报价时效数据"2h→10min"在论文中出现于两处：
- P129：归因于"数字化系统已实现"
- P193：归因于"AI智能体有望实现"

分析结论：同一对数字被归因于两个不同阶段，逻辑矛盾。

### 修正策略

- **增量表述**：将后一处改为"10min→2min"，避免重复归因
- **删除重复**：仅保留一处（选择归因最准确的位置）
- **标注冲突**：在暂不确定时标注"数据归因需确认"

### 输出格式

```
P014引言 ✓无影响
P026节标题 ✓无影响
P129数据源 ✗需修改（归因重写）
P193数据源 ✗需修改（改为增量表述）
P217结论 ✓无影响
```
