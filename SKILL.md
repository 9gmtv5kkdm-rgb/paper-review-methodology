# 学术内容全面审查方法论（Content Review Methodology）

## 相关技能与路由
- 要**审稿/逐段审查法（33类/7体系）**：用本技能。
- 要**多版本追踪/变更日志**：用 `paper-version-manager`。
- 要**投稿全流程（Cover Letter/Rebuttal/清单）**：用 `submission-helper`。
- 要**查重/降重路径**：用 `plagiarism-precheck`。

33类独立审查方法论，覆盖学术论文从宏观评估到逐段执行的完整审查链条。

## Quick Start（5 分钟快速上手）

> 你不需要先读完 33 类方法。记住这条最小闭环即可开工：

**最小审查闭环（5min）**

1. **定身份与范围**：以「独立审稿人」视角，明确审查维度（创新点 / 结构 / 数据 / 引用 / 语言）。
2. **跑框架**：用第 1 体系「四维度评估」给宏观打分，再用第 5 体系做数据合规与脱敏筛查。
3. **出报告**：按「分数 + 问题分级(P0–P3) + 修改建议」三段输出，绝不替代正式同行评议。
4. **要深挖**：逐段比对、跨段一致性、引用重编号等见对应 `references/*.md`。

**一句话示例**：用户说「帮我审一下这篇论文的创新点」→ 你以审稿人视角给出：创新点评分 + 发现的 P0/P1 问题 + 可操作修改建议。完整话术与分级见 `references/macro-assessment.md` 与 `references/problem-grading.md`。

## 典型话术模式

### 开篇句式
- 「以xxx身份审查/审阅xxx」
- 「从xxx维度对xxx进行系统性审查」
- 「发现xxx项问题：P0级x项/P1级x项/P2级x项/P3级x项」

### 评判句式
- 「该问题的性质是xxx」「严重程度为P0级」
- 「推荐方案A（xxx），替代方案B（xxx）」
- 「这一发现与前文xxx形成交叉验证」
- 「三个维度均不合格 → 建议xxx」

### 收束句式
- 「审查完成。核心结论：xxx」
- 「修正硬伤后预计：xx-xx分」
- 「等待用户确认/下达修改指令后执行」
- 「综合评分：X维度的加权总分」

### 自纠机制
- 「撤销：1项（AI痕迹过度敏感，撤销判定）」
- 「降级：1项（长句→P3，不作为独立修改项）」
- 「此前的xxx判定有误，现更正为xxx」

## 审查流程建议

1. **首轮** → 四维度评估 + P0-P3分级 + AI语言审查框架 → 确定整体质量和优先级
2. **专项轮** → 插图引用审查 + 参考文献比对 + 中英文摘要对称 + 模块一致性
3. **数据轮** → 数据真实性验证 + 双抱归因分析 + 脱敏筛查 + 文献真实性交叉验证 + 规范性文件引用检测 + 全文涟漪扫描
4. **对标轮** → 版本交叉引用审计 + 决策/需求满足度审计
5. **终审轮** → P0逐项修正审计 + 逐段比对 + 引用重编号 + 评分追踪

## 评分公式

加权评分：内容完整性×0.3 + 逻辑完整性×0.25 + 逻辑闭环×0.25 + AI检测评估×0.2

优质线：85分（具备送审条件）

## 质量标准与红线

### 四维质量评分

| 维度 | 定义 | 优秀(3分) | 达标(2分) | 不及格(0分) |
|------|------|-----------|-----------|------------|
| 审查覆盖率 | 方法选择是否匹配问题 | 完全匹配 | 大部分覆盖 | 方法不当 |
| 分级准确性 | P0-P3分级是否合理 | 全部准确 | 1处偏差 | 多处偏差 |
| 评分可复现性 | 同一论文两次评分是否一致 | Δ≤3分 | Δ≤8分 | Δ>8分 |
| 建议可操作性 | 修改建议是否具体可执行 | 逐条可操作 | 需再解读 | 太模糊 |

加权总分 = Σ(维度分) / (4×3) × 100
优质线：≥75分 ｜ 合格线：≥50分 ｜ 重做线：<50分

### 质量红线

- 🔴 不修改论文内容——只提供审查诊断报告和修改建议
- 🔴 每项P0问题必须有明确的修改指示
- 🔴 自纠机制必须记录在审查报告中（撤销/降级记录）



## 诊断系统

### 快车道：案例库

| 模式名 | 症状 | 修正操作 | 示例 |

## 联网规则

1. 当需要最新审查标准时，搜索特殊审查需求时最新标准
2. 当需要学科特定规范时，搜索非标准学科时其审查惯例
3. 任何联网获取的信息必须注明来源
4. 联网结果仅供辅助决策，不替代用户自身判断
5. 如果联网失败，基于已有知识和模板继续工作并告知用户
6. 超出知识范围时优先联网搜索而非猜测
