# 执行级方法论（3类）

## #27 六项修改明细清单

### 格式规范

每条修改包含：

```
修改编号 + 精确段落编号（如P016）
+ 具体文字操作（删除/增补/重写/改题注）
+ 效果描述
```

### 操作类型词汇

| 类型 | 说明 | 示例 |
|------|------|------|
| 增补 | 添加新段落/句子/数据 | 在P036后添加衔接段 |
| 删除 | 移除冗余/错误内容 | 删除P089重复段 |
| 重写 | 保留原意改表述 | 重写P045的AI语言段落 |
| 改编号 | 修正图/表/章节编号 | 图3→图4 |
| 追加衔接句 | 添加桥接过渡 | 在P112后添加"基于以上原则..." |
| 图题简化 | 精简图片标题 | "图1 溯源系统四层闭环架构及十大核心模块"→"图1 溯源系统四层闭环架构" |
| 图片更换 | 替换/更新插图 | 用新版架构图替换P027图片 |

### 清单示例

```
修改一（P016归属修正）：将设备管理从Ch3移至Ch4，修正三层归属描述
修改二（P027图1更换）：更换为四层闭环架构新图
修改三（P052 RBAC重写）："班组/个人双维度"→"10角色标准RBAC权限模型"
修改四（P129/P193双抱归因修复）：报价时效改为增量表述10min→2min
修改五（P217结论补全）：新增质量处理周期↓70%、工序交接↓80%
修改六（Ch2.1补充）：行业宏观数据[需补充]→专家提供后填入
```

## #28 修改前后逐段比对

### 方法

1. 提取当前文档全文到txt（逐段编号）
2. 对照修改清单逐段检查执行状态
3. 标注执行结果

### 标注符号

| 符号 | 含义 |
|:----:|------|
| ✓ | 已修改完毕 |
| ✗ | 待修改 |
| • | 无需修改 |
| ⚠ | 修改有偏差需复查 |

### 输出格式

```
P014 ✓ 已修改
P016 ✗ 待修改（归属未修正）
P020 • 无需修改
P027 ✓ 已修改
...
```

### 操作建议

- 从末尾向前操作避免段落索引漂移
- 修改一处的段落索引变化可能影响后续比对，建议分批次比对
- 每批次<10处修改后重新提取全文

## #29 P0硬伤逐项修正审计

### 审计流程

```
定位P0问题 → 修正 → 涟漪扫描验证 → 标记完成
```

### 检查清单模板

```
□ 新增物料/仓库模块（填补十大模块遗漏）
□ 新增工艺/技术模块（填补论文完全缺失）
□ RBAC权限模型重写（10角色标准RBAC）
□ 双抱归因修复（增量表述10min→2min）
□ 引用缺失修复（图5/表1/表2/表3各至少1次引用）
□ 参考文献零引用修复
□ 编造数据替换为[需补充]占位符
□ 架构冲突解决（选定唯一架构方案）
□ 每项修正后用涟漪扫描验证
```

### 涟漪扫描验证

每次P0修正后立即启动涟漪扫描（见#15），确认：
- 修正是否正确执行
- 是否引入新问题
- 是否有需要同步修改的关联段落

### 完成标记

```
✅ P0-001 修复完成，涟漪扫描通过
✅ P0-002 修复完成，涟漪扫描通过
✅ P0-003 修复完成，涟漪扫描通过
⏳ P0-004 修复中...
```
